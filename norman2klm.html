Norman's Model

Example: Switching on a desk lamp
Phases Stages Example
Execution Establishing the goal. Need more light while reading.
Forming the intention. Switch on the desk lamp.
Specifying the action sequence Reach over and press the lamp switch.
Executing the action Press the lamp switch.
Evaluation Perceiving the system state. Observe if the light is on.
Interpreting the system state If light doesn’t come on → bulb blown or lamp unplugged.
Evaluating Outcome If light is sufficient → goal achieved; if not → formulate new goal (e.g., switch on ceiling light).

Reference: (Dix et al., Chapter 3 – The Interaction / Model of Interaction) page no 126

GOMS Model

GOMS (Goals, Operators, Methods, Selection) model helps us analyze the steps a user takes to achieve a goal. Let’s take an example of minimizing a window on a computer.

GOMS Example

Method 1: CLOSE-METHOD

  • Goal: ICONIZE-WINDOW
    • USE-CLOSE-METHOD
      • Move mouse to window header
      • POP-UP menu
      • Click over close option

Method 2: L7-METHOD

  • Goal: ICONIZE-WINDOW
    • USE-L7-METHOD
      • Press L7 key

Selection Rule

  • User Sam:
    • Use CLOSE-METHOD unless another rule applies
    • if the application is 'blocks', use L7-METHOD

Reference: (Dix et al., Chapter 12 – Cognitive Models / Goal and task hierarchies) page no 422

Keystroke-Level Model

Worked Example: Correcting a Single Character

Example task: the user notices a single-character error in a mouse-based editor, deletes it, retypes it, and returns to the insertion point.

  1. Move hand to mouse
  2. Position mouse after bad character
  3. Return to keyboard
  4. Delete character
  5. Type correction
  6. Reposition insertion point

One possible KLM operator sequence:

            H[mouse]
            P B[LEFT]
            H[keyboard]
            M K[DELETE]
            K[char]
            H[mouse] M P B[LEFT]

This sequence illustrates how KLM models only execution-level actions. High-level planning, learning, and interpretation are not represented.

Reference: (Dix et al., Chapter 12 – Cognitive Models / Physical and Device Models) page no 436

Key Takeaways

The Keystroke-Level Model consists of seven primitive operators (K, B, P, H, D, M, R) used to predict execution time for expert, error-free interaction, and is a low-level instance of the GOMS framework.

KLM has only seven operators because it models execution, not thinking about what to do.
