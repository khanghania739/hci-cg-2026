HCI Models

Don Norman's Model

Don Norman, a pioneer in Human–Computer Interaction, proposed the action Cycle to explain how users interact with systems.

Execution and Evaluation Cycle:

  • Execution Phase: Focused on actually performing the planned actons.
  • Evaluation Phase: Focused on checking if the goal was achieved based on system feedback.

Gulfs

The gaps between what the user wants to do and what the system actually does.

  • Gulf of Execution: When the user wants to perform an action but is not sure how to do it on the system.
  • Gulf of Evaluation: When the system responds but the user cannot easily understand if their action worked.

7 Stages of Norman's Model

The Execution-Evalaution Cycle
Phases Stages Example
Execution Establishing the goal. You decide you need more light while reading.
Forming the intention. You can plan to switch on the desk lamp yourself.
Specifying the action sequence You plan to reach over and press the lap switch.
Executing the action You press the lamp switch.
Evaluation Perceiving the system state. You see if the lamp turns on.
Interpreting the system state you understand what happened (lamp works or not)
Evaluating the system state You check if there is enough light now

Reference: (Dix et al., Chapter 3 – The Interaction / Model of Interaction) page no 126

GOMS Model

GOMS is a task-analysis model used in HCI to predict how users perform tasks. It breaks down tasks into:

Goal: The result or outcome that the user wants to achieve by using the system.

Operators: The smallest actions a user can perform, like clicking, typing, pointing, or moving the mouse.

  • Keystroke: Pressing a key.
  • Pointing: Moving the mouse to a target.
  • Clicking: Clicking a button or menu item.
  • Selecting: Choosing an item from a list.
  • Mental Preparation: Thinking about what to do next.
  • System Response Wait: Waiting for the computer to respond.

Methods: A series of steps or actions that the user follows to reach the goal.

Selection Rules: How the user decides which method to use when there are multiple options.

This model helps in estimating task time and optimizing user efficiency.

Reference: GOMS (Dix et al., Chapter 12 – Cognitive Models / Goal and task hierarchies) page no 422

KLM (Ketstroke-Level Model)

KLM is a predictive model for estimating the time it takes an expert user to perform a task without errors. It assigns time values to keystrokes, pointing, and other basic actions, allowing designers to evaluate interface efficiency quantitatively.

Phase

  • Acquisition: The user learns and understands the task by forming a mental picture of what needs to be done.
  • Execution: The user performs the task using the tools and functions provided by the system.
KLM operators
Operators Name Meaning Example
K Keystroking Pressing a key (including modifiers) Typing a character or pressing Delete
B Button press Pressing a mouse button Left-click
P Pointing Moving pointer to a target Move cursor to an icon
H Homing Switching hand between devices Keyboard → mouse
D Drawing Drawing lines using a pointing device Dragging to draw a shape
M Mental preparation Preparing for a physical action Pause before clicking the correct control
R System response Waiting for system feedback (if required) Application loading

Reference: KLM (Dix et al., Chapter 12 – Cognitive Models / Physical and Device Models) page no 436
